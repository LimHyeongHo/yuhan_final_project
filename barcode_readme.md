# 📚 도서 바코드 스캐너 (YU-BOOK MVP)

스마트폰 카메라를 이용하여 전공책의 바코드(ISBN)를 스캔하고, **알라딘 오픈 API**를 통해 책의 상세 정보(제목, 저자, 정가, 표지 등)를 실시간으로 조회하는 모바일 웹 기반 애플리케이션입니다. 

본 프로젝트는 유한대학교(Yuhan University) 최종 프로젝트의 프론트엔드 베타 테스트(MVP) 버전으로, 백엔드 서버 없이 자바스크립트 단독으로 외부 API와 통신하여 데이터를 렌더링하도록 구현되었습니다.

<br>

## ✨ 주요 기능 (Key Features)

* **📱 모바일 최적화 스캐너**
  * `html5-qrcode` 라이브러리를 활용하여 스마트폰의 후면 카메라(`environment`)를 강제 활성화합니다.
  * 불필요한 UI를 제거하고 사용자 정의 버튼과 커스텀 스캔 영역(300x150)을 적용했습니다.
* **🛡️ 바코드 오인식 방어 로직 (필터링)**
  * 카메라 초점이 흔들려 잘못된 숫자가 인식되는 것을 막기 위해, 국제 표준 도서 번호(ISBN-13) 규칙에 따라 **13자리**이면서 **`978` 또는 `979`로 시작하는 바코드만 처리**하도록 예외 처리가 적용되어 있습니다.
* **🌐 CORS 에러 우회 통신**
  * 백엔드가 없는 현재 환경에서 브라우저의 CORS 정책을 우회하기 위해 `corsproxy.io` 프록시 서버를 적용하여 안정적으로 알라딘 API 데이터를 가져옵니다.
* **🛡️ 방어적 프로그래밍 (Data Handling)**
  * 줄거리나 페이지 수 데이터가 누락된 도서를 스캔했을 때 프로그램이 멈추지 않도록 옵셔널 체이닝(`?.`) 및 OR 연산자(`||`)를 활용한 데이터 안전망이 구축되어 있습니다.

<br>

## 🛠 기술 스택 (Tech Stack)

* **Frontend:** HTML5, CSS3, Vanilla JavaScript (ES6+)
* **Library:** [html5-qrcode](https://github.com/mebjas/html5-qrcode) (바코드 인식 코어 엔진)
* **Open API:** 알라딘 도서 검색 API (`ItemLookUp`)

<br>

## 🔄 시스템 동작 흐름 (Logic Flow)

1. **스캔 시작:** 사용자가 [바코드 스캔하기] 버튼 클릭
2. **카메라 렌더링:** 스마트폰 후면 카메라 활성화 및 스캔 영역 표시
3. **바코드 인식 & 검증:** 스캔된 데이터가 EAN-13 규격(978/979)에 맞는지 유효성 검사
4. **API 요청:** 검증된 ISBN 번호를 바탕으로 알라딘 API URL 조립 및 프록시(`corsproxy.io`) 경유 Fetch 요청
5. **DOM 조작:** 응답받은 JSON 데이터를 파싱하여 HTML 화면에 책 표지, 가격(콤마 포맷팅), 줄거리 등을 즉시 렌더링 (동시에 카메라는 백그라운드에서 자동 종료됨)

<br>

## 🚀 실행 방법 (Usage)

본 프로젝트는 정적 웹 페이지로 구성되어 있어 별도의 빌드 과정이 필요하지 않습니다.

**1. 로컬 환경 테스트 (PC & 모바일)**
VS Code의 `Live Server` 확장을 통해 `barcode_test.html`을 실행한 뒤, 포트 포워딩(Port Forwarding) 기능을 통해 스마트폰 브라우저에서 로컬호스트로 접속하여 카메라를 테스트할 수 있습니다.

**2. GitHub Pages 배포 확인**
코드가 GitHub 저장소의 `main` 브랜치에 푸시되면 GitHub Pages를 통해 자동 배포됩니다.
👉 **Live Demo:** `https://limhyeongho.github.io/yuhan_final_project/Barcode/barcode_test.html` (스마트폰 브라우저 접속 권장)
** 만약 접속이 안 될 경우** :`https://limhyeongho.github.io/yuhan_final_project/Barcode/barcode_test.html?v=1` 혹은 v=2 로 변경 후 접속하면 가능합니다.

## 💡 참고 (개발자 주요 메모)

현재 소스 코드에는 향후 백엔드(Java/Spring) 및 실제 서비스 연동을 대비하여 `/// [*]` 기호로 중요한 개발 이정표가 주석 처리되어 있습니다. 프로젝트 고도화 시 다음 사항들을 우선적으로 반영할 예정입니다.

* **1. API 보안 및 백엔드 연동 (약 74번째 줄)**
  * `/// [*] api 키값과 퍼블릭 키값은 나중에 백엔드로 저장`
  * **위치:** `ScanSuccess` 함수 내부 (URL 조립 단계)
  * **설명:** 현재 프론트엔드 환경에서 테스트를 위해 하드코딩된 알라딘 API 키(`ALADIN_API_KEY`)는 보안상 취약하므로, 향후 백엔드 서버 구축 시 서버 내부 환경변수(`.env` 등)로 숨겨서 관리하도록 아키텍처를 변경해야 합니다.

* **2. 데이터베이스(DB) 연동 (약 100번째 줄)**
  * `/// [*] DB에 저장할 경우 하단 데이터를 DB에 연결하면 됨.`
  * **위치:** `LookUpBookInfo` 함수 내부 (API 데이터 객체 할당 단계)
  * **설명:** API를 통해 성공적으로 불러온 책의 상세 정보(제목, 저자, 정가, 표지 등)를 화면에 띄우는 것에 그치지 않고, YU-BOOK 자체 데이터베이스에 저장(POST 요청)할 때 이 블록의 데이터를 백엔드로 전송하도록 로직을 추가할 예정입니다.

* **3. 카메라 방향(Facing Mode) 전환 옵션 (약 134번째 줄)**
  * `/// [*] 나중에 만약 핸드폰을 놔두고 작업할 경우 전면 카메라로 수정함.`
  * **위치:** `html5QrCode.start` 스캐너 실행 설정 부분
  * **설명:** 현재는 사용자가 스마트폰을 들고 전공책을 찍는 상황을 가정하여 후면 카메라(`{ facingMode: "environment" }`)를 기본으로 사용하고 있습니다. 향후 스마트폰을 거치대에 고정해 두고 책을 가져다 대는 키오스크 방식 등으로 작업 환경이 변경될 경우, 이 설정을 전면 카메라(예: `"user"`)로 손쉽게 수정할 수 있습니다.
