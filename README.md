# 🛒 공동구매 예외처리 데모

공동구매 시스템에서 마감 처리 시 발생하는 정상/예외 시나리오를 시연하는 Spring Boot 데모 프로젝트입니다.

---

## 기술 스택

- Java 17
- Spring Boot 4.0.5
- Spring Data JPA
- H2 인메모리 데이터베이스 (data.sql에 수동으로 입력하여 저장했습니다.)
- Thymeleaf


---

## 시나리오 설명

### 시나리오 A — 정상 처리
- 최소 인원 2명 / 참여자 3명
- 마감 시 미입금자(동동동) 제외
- 참여자 3명 → 2명, 금액 재계산 (12,500원 → 18,750원)
- 상태: 모집중 → 발주대기 → 발주진행

### 시나리오 B — 예외 처리
- 최소 인원 2명 / 참여자 2명
- 마감 시 미입금자(동동동) 제외
- 참여자 2명 → 1명, 최소 인원 미달
- 상태: 모집중 → 실패, 결제자 환불 처리

---

## 관리자 액션

| 액션 | 설명 |
|------|------|
| 발주 확정 | 발주대기 → 발주진행, 구매 이력 저장 |
| 마감 연장 | 미입금자 퇴출 + 마감시간 1일 연장 |
| 실패 확정 | 공동구매 실패, 결제자 전원 환불 |
| 데모 초기화 | 초기 데이터로 리셋 |
