# 💬 WebSocket 실시간 채팅 데모

이 프로그램은 WebSocket의 핵심 원리인 **실시간 양방향 통신**을 Spring Boot와 React로 구현한 채팅 데모입니다.
대학생 공동구매 마켓 플랫폼에서 공동구매 참여자들이 실시간으로 소통할 수 있는 채팅 기능을 시연합니다.

---

## 📌 주요 기능

1. **실시간 채팅**
   - SockJS + STOMP 프로토콜 기반으로 서버-클라이언트 간 실시간 양방향 통신 구현
   - 메시지 전송 즉시 채팅방 내 모든 참여자에게 브로드캐스트

2. **참여자 목록 실시간 관리**
   - 새 유저 입장 시 서버에서 현재 참여자 전체 목록을 수신하여 즉시 반영
   - 브라우저 종료 등 세션 끊김 감지 시 참여자 목록에서 자동 제거

3. **입장 / 퇴장 알림**
   - 유저 입장 및 퇴장 시 채팅창 중앙에 시스템 알림 메시지 표시

4. **메시지 UI 구분**
   - 내 메시지는 오른쪽(#1a7a6e 색상), 상대방 메시지는 왼쪽(#2a2a2a 색상)으로 표시

---

## ⚙️ 실행 방법

### 시스템 요구사항
- Java 17 이상
- Node.js 18 이상
- npm 9 이상

---

### Step 1. 백엔드 실행 (Spring Boot)

1. IntelliJ IDEA에서 `Chatting` 폴더를 엽니다.
2. `src/main/java/com/springboot/chatting/ChattingApplication.java` 를 찾습니다.
3. 상단 초록색 ▶️ 버튼을 클릭하여 실행합니다.
4. 하단 콘솔에 아래 메시지가 출력되면 성공입니다.
```
Started ChattingApplication in X.XXX seconds
```
> 백엔드 서버는 **http://localhost:8080** 에서 실행됩니다.

---

### Step 2. 프론트엔드 실행 (React)

터미널을 열고 아래 명령어를 순서대로 입력합니다.

```bash
cd frontend
npm install
npm start
```

브라우저가 자동으로 열리며 **http://localhost:3000** 으로 접속됩니다.

> ⚠️ 반드시 백엔드(Spring Boot)를 먼저 실행한 후 프론트엔드를 실행하세요.

---

### Step 3. 테스트 방법

1. 브라우저에서 `http://localhost:3000` 접속
2. 아래 테스트 계정 중 하나를 선택하여 입장

| 계정 | 역할 |
|---|---|
| admin1 | 구매자 |
| admin2 | 판매자 |
| admin3 | 관리자 |

3. 실시간 채팅 테스트를 위해 **크롬 시크릿 창** (Mac: `Cmd + Shift + N`) 을 추가로 열어 다른 계정으로 입장합니다.
4. 두 창에서 메시지를 주고받으며 실시간 채팅을 확인합니다.

## 💡 기술적 특징

- **STOMP 프로토콜 활용**: 단순 WebSocket이 아닌 STOMP를 적용하여 메시지 목적지(`/topic/public`)를 지정하고, 다수의 구독자에게 효율적으로 메시지를 브로드캐스트합니다.
- **SockJS 폴백 처리**: WebSocket을 지원하지 않는 환경에서도 SockJS가 자동으로 대체 통신 방식으로 연결을 유지합니다.
- **참여자 목록 서버 관리**: `ChatController`의 `static Set`으로 현재 접속 중인 참여자를 서버에서 직접 관리하며, 새 유저 입장 시 전체 목록을 함께 전송합니다.
- **세션 기반 퇴장 감지**: `WebSocketEventListener`가 WebSocket 세션 종료 이벤트(`SessionDisconnectEvent`)를 감지하여 브라우저를 닫아도 자동으로 퇴장 처리합니다.
- **MessageType ENUM**: `CHAT`, `JOIN`, `LEAVE` 세 가지 메시지 타입을 구분하여 채팅 메시지와 시스템 알림을 분리 처리합니다.

## ⚠️ 주의사항

- 본 데모는 DB 연동 없이 서버 메모리에서 참여자를 관리하므로, Spring Boot 재시작 시 참여자 목록이 초기화됩니다.
- 테스트용 로컬 환경에서만 동작하며 실제 배포 환경에서는 HTTPS 및 보안 설정이 추가로 필요합니다.

## 🔮 추가 구현 예정 기능

> 현재 데모는 WebSocket 실시간 통신 원리를 시연하기 위한 프로토타입입니다.
> 이후 실제 서비스 수준으로 아래와 같이 단계적으로 확장할 예정입니다.

---

### 1. 채팅 유형 분리 — 단톡방 + 1:1 문의 채팅

**① 공동구매 단톡방 (GROUP)**
- 참여 버튼 클릭 시 해당 공동구매 단톡방에 자동 입장
- 참여자 전원 + 판매자가 함께 소통하는 공간
- 판매자가 발주 확정, 배송 시작 등 진행 상황을 전체 공지

**② 판매자 1:1 문의 채팅 (PERSONAL)**
- 상품 페이지에서 1:1 문의 버튼 클릭 시 개인 채팅방 생성
- 배송지, 사이즈, 옵션 등 개인 문의용
- 기존에 같은 판매자와의 채팅방이 있으면 기존 방으로 바로 입장

---

### 2. DB 연동 — 채팅 기록 영구 저장

현재는 서버 재시작 시 모든 대화 내역이 초기화됨.

- 채팅방 입장 시 REST API로 이전 메시지 페이징 로드
- 이후 새 메시지는 기존과 동일하게 WebSocket으로 실시간 수신
- 두 방식을 혼합한 하이브리드 구조로 구현 예정

---

### 3. 읽음 / 안읽음 처리 — 카카오톡 스타일

- 1:1 채팅 : 상대방이 읽지 않으면 말풍선 옆 **1** 표시, 읽으면 사라짐
- 단톡방 : 아직 읽지 않은 참여자 수를 숫자로 표시
- 채팅방 입장 시 읽음 처리되어 숫자 초기화

메시지마다 `is_read` 컬럼을 두는 대신, 참여자별 **`last_read_message_id`** 로 관리해 단톡방에서도 정확한 읽음 수 계산 가능.
---

### 4. 채팅방 목록 UI

- 단톡방 + 1:1 채팅 한 목록에서 통합 확인
- 최근 메시지 기준 내림차순 정렬
- 상품명 또는 상대방 이름, 마지막 메시지 미리보기, 전송 시각 표시
- 안읽은 메시지 있으면 뱃지로 표시, 새 메시지 수신 시 실시간 갱신

---

### 5. 실제 배포 환경 전환 — HTTPS 및 보안 설정

현재는 로컬 환경(`http://localhost`)에서만 동작.

- **HTTPS 전환** : 배포 시 SSL 인증서 적용, `ws://` → `wss://` 프로토콜로 전환
- **CORS 설정** : 현재는 모든 출처 허용 중 → 배포 시 실제 도메인만 허용하도록 제한
- **WebSocket 인증** : 현재는 닉네임 입력만으로 입장 가능 → JWT 토큰 기반 인증으로 전환
